# Attribution

This project creates machine-generated personal-name attestation indexes from
Latin inscription corpora. It is a derived, computational resource by Michael
A Bosse, not an official release of EDCS, LIRE, EDH, Trismegistos, CIL ACE,
Ubi Erat Lupa, DARE, or any other source database.

## Recommended attribution

When using the derived data, cite the project release and the source datasets:

> Michael A Bosse, Roman Names: machine-generated personal-name attestations
> from Latin inscriptions. Derived from EDCS 2022 and LIRE v3.0. Generated with
> the `metiscus/roman-names` pipeline.

When a Zenodo DOI is minted for this project, cite that DOI in place of the
repository-only reference.

## Source datasets

### EDCS 2022

- Dataset: EDCS
- Version used: 2.0, aggregated 2022-09-12
- Zenodo DOI: https://doi.org/10.5281/zenodo.7072337
- Zenodo license metadata: CC BY 4.0
- Creator listed on Zenodo: Petra Heřmánková, Aarhus University
- Original database: Epigraphik-Datenbank Clauss-Slaby (EDCS),
  https://db.edcs.eu/
- Fields used include EDCS identifiers, raw inscription text, province,
  findspot/place metadata, date fields, and inscription type.

### LIRE v3.0

- Dataset: Latin Inscriptions of the Roman Empire (LIRE)
- Version used: v3.0, published 2023-10-11
- Zenodo DOI: https://doi.org/10.5281/zenodo.8431452
- Zenodo license metadata: CC BY 4.0
- Creators listed on Zenodo: Vojtěch Kaše, Petra Heřmánková,
  Adéla Sobotková, Aarhus University
- Project repository: https://github.com/sdam-au/LI_ETL
- Fields used include coordinates, dates, interpretive text, publication
  references, external identifiers, and structured `people` data used for
  validation.

LIRE aggregates material from EDCS and the Epigraphic Database Heidelberg
(EDH). LIRE's own disclaimer notes that original data is provided by the
third-party source indicated in its metadata and recommends checking accuracy
against the primary source or editio princeps.

## Generated annotations

Name extraction, name-component classification, social-status labels,
fragmentary flags, non-person flags, heuristic clusters, summaries, and English
translations are generated by this project. They should be treated as
machine-generated scholarly aids requiring verification against the inscription
and relevant publications.

Clusters are heuristic possible same-person groups, not confirmed
prosopographical identities.
